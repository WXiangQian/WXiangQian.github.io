EasyWeChat在laravel框架中的使用技巧

laravel框架实战：
EasyWeChat

EasyWeChat公众号配置

protected $wechat;

public function __construct ()
{
    $this->wechat = app('wechat.official_account');  //微信公众号配置
}

报错:Failed to cache access token.

报错:Failed to cache access token.的原因大部分是：token缓存文件没写成功。

在使用wechat方法的上面加入以下代码:

 $this->wechat->cache = new CacheBridge(app('cache.store'));

获取公众号临时二维码

$res = $this->wechat->qrcode->temporary('WXiangQian', 6 * 24 * 3600);

公众号模板消息推送

$this->wechat->template_message->send([
    'touser' => $this->message['FromUserName'],
    'template_id' => 'template_id',// 模板消息id
    'url' => 'https://wxiangqian.github.io/', // 要跳转的url
    'data' => [
        "first" => "欢迎来到本公众号",
        "keyword1" => '你好，WXiangQian', // 必须为keyword
        "keyword2" => "欢迎来到本公众号",
        "keyword3" => "恭喜你成功入坑",
        "remark" => "小提示：期待您的持续关注", //备注
    ]
]);

公众号服务端的使用

在微信开发中主要是负责接收用户发送过来的消息，还有用户触发的一系列事件

$app->server->push(function ($message) {
    // $message['FromUserName'] // 用户的 openid
    // $message['MsgType'] // 消息类型：event, text....
    return "您好！欢迎使用 EasyWeChat";

$response = $app->server->serve();

// $response 为 `Symfony\Component\HttpFoundation\Response` 实例

            break;

MsgType Event 事件类型详解

扫描带参数的二维码事件

EventKey 事件KEY值，比如：qrscene_123123，qrscene_为前缀，后面为二维码的参数值
Ticket 二维码的 ticket，可用来换取二维码图片

// （如：subscribe(订阅)、unsubscribe(取消订阅) ...， CLICK 等）
if($message['MsgType'] == 'event') {  //收到事件消息
    switch ($message['Event']) {
        case 'CLICK':          //点击事件
            break;
        case 'subscribe':      //订阅
            $event_info = substr($this->message['EventKey'], 8);
            $event_arr = explode('_', $event_info);
            if (!empty($event_arr) && $event_arr[0] == 'test') 
                // test逻辑
            }else{
                // 关注公众号逻辑
                return $this->subscribe($event_arr);
            }
            break;
        case 'unsubscribe':    //取消订阅
            return $this->unSubscribe();
            break;
        case 'SCAN':           //已关注的情况下扫描二维码
            $this->scan();
            break;
    }
}

小程序报错：Identifier “subscribe_message” is not defined.

报错的原因是当前的laravel-wechat版本过低，需要更新，小程序订阅消息是后续新增的功能，所以说低版本用不了订阅消息。小程序的模板消息已停用，后续都会接入订阅消息。

小程序订阅消息推送

$send_data = [
    'template_id' => 'template_id', // 所需下发的订阅模板id
    'touser' => 'openid',     // 接收者（用户）的 openid
    'page' => '/pages/index,       // 点击模板卡片后的跳转页面，仅限本小程序内的页面。支持带参数,（示例index?foo=bar）。该字段不填则模板无跳转。
    'data' => [         
    // 模板内容，格式形如 { "key1": { "value": any }, "key2": { "value": any } }
        'thing2' => [
            'value' => '课程标题',
        ],
        'character_string6' => [
            'value' => '1/1',
        ],
        'thing7' => [
            'value' => '恭喜你已成功解锁这个课程',
        ],
    ],
];
// 非线上环境
if (env('APP_ENV') != 'production') {
    $send_data['miniprogramState'] = 'developer';
}
$this->app->subscribe_message->send($send_data);
